## Overview

Cipher is a cyber threat attribution and analysis system designed for homeland security intelligence operations. The platform provides threat analysis, attribution, and risk assessment capabilities for national security and critical infrastructure protection.

### Key Capabilities

- **Cyber Threat Intelligence**: Real-time threat detection and analysis
- **IOC Tracking**: Automated collection from multiple sources (OTX, Abuse.ch, PhishTank, NVD)
- **Threat Attribution**: Identify and attribute cyber threats to actors using MITRE ATT&CK
- **Anomaly Detection**: Zero-day detection using autoencoders and Isolation Forest
- **Network Visualization**: Interactive threat actor network graphs
- **Campaign Correlation**: Link related IOCs into campaigns

### Use Cases

- Zero-day threat detection and analysis
- Threat actor attribution using MITRE ATT&CK
- IOC correlation and campaign identification
- Multi-source threat intelligence integration
- Network visualization and threat graph analysis

## Quick Start

- Python 3.11+
- Docker and Docker Compose
- (Optional) API keys for OTX, PhishTank, NVD (improves rate limits)

### Installation

1. **Clone the repository**
   ```bash
   git clone https://github.com/reichert-sentinel-ai/cipher-threat-tracker
   cd cipher-threat-tracker
   ```

2. **Set up environment variables**
   ```bash
   cp .env.example .env
   # Edit .env with your API keys (optional)
   ```

3. **Start with Docker Compose**
   ```bash
   docker-compose up -d
   ```

   This starts:
   - FastAPI API (http://localhost:8000)
   - Streamlit Dashboard (http://localhost:8501)
   - Elasticsearch (http://localhost:9200)
   - Neo4j (http://localhost:7474)

4. **Access services**
   - API Documentation: http://localhost:8000/docs
   - Dashboard: http://localhost:8501
   - Neo4j Browser: http://localhost:7474

## Usage

### Collect IOCs

```python
from src.collectors.ioc_orchestrator import IOCOrchestrator
from src.utils.elastic import ElasticsearchClient

# Collect from all sources
orchestrator = IOCOrchestrator()
iocs = orchestrator.collect_all(limit_per_source=100)

# Index in Elasticsearch
es_client = ElasticsearchClient()
es_client.create_index()
es_client.bulk_index(iocs)
```

### Check IOC

```bash
curl -X POST "http://localhost:8000/api/v1/ioc/check" \
  -H "Content-Type: application/json" \
  -d '{"ioc_value": "192.0.2.1", "ioc_type": "ip"}'
```

### Detect Anomalies

```python
from src.models.autoencoder import AnomalyDetector, TrafficAutoencoder
import torch

# Load trained model
model = TrafficAutoencoder()
detector = AnomalyDetector(model)

# Fit threshold on normal data
normal_data = torch.randn(100, 512)
detector.fit_threshold(normal_data)

# Detect anomalies
test_data = torch.randn(10, 512)
is_anomaly, scores = detector.detect(test_data)
```

## API Endpoints

### IOC Endpoints
- `POST /api/v1/ioc/check` - Check if IOC is known threat
- `GET /api/v1/ioc/search` - Search IOCs
- `POST /api/v1/ioc/collect` - Trigger IOC collection

### Threat Endpoints
- `GET /api/v1/threats` - Get active threats
- `GET /api/v1/threats/stats` - Get threat statistics

### Detection Endpoints
- `POST /api/v1/detect/anomaly` - Detect anomalies
- `POST /api/v1/detect/classify` - Classify IOC threat type

### Network Endpoints
- `GET /api/v1/network` - Get threat actor network graph
- `GET /api/v1/actors` - Get threat actors

See full API documentation at http://localhost:8000/docs
